Method Shipping Scheduling

If the Method Availability Scheduling is set to “Once Only” or “Recurring” then additional options for specifying when an order is expected to be shipped become available for use.

Once Only

If the Method Availability Scheduling option is set to “Once Only” then the Shipping Date/Time option can be used to set the date/time when shipping for this method is expected to take place.

The shipping date will be recorded alongside the standard details for the order, and information about the shipping date can be displayed in the method's title using “conversion specifiers”, as detailed in Appendix A.

Recurring

If the Method Availability Scheduling option is set to “Recurring” then Shipping Scheduling can be enabled by setting the Method Shipping Scheduling option to “Shipping On A Regular Weekday”.

The options available then depend on the way the method has its “Recurring Weekly” Availability Scheduling configured. There are two possible situations:

  1. If both a Weekly Start Day/Time and a Weekly Cut-off Day/Time were specified in the Method's Availability Scheduling then the only option to be configured is the Weekly Shipping Day/Time. This is the Day of the week and Time of that day when orders using the method are expected to be shipped.

    The shipping date will be recorded alongside the standard details for the order, and information about the shipping date can be displayed in the method's title using “conversion specifiers”, as detailed below.

  2. If just the Weekly Cut-off Day/Time has been specified in the Method's Availability Scheduling then the Number of Weeks to Show configuration option can be used to specify how many weeks in advance to show the method as being available for.

    The Weekly Shipping Day/Time option is used to specify the Day of the week and Time of that day any order using that method will be shipped.

    The method is then repeated as a shipping option for the number of weeks specified in the Number of Weeks to Show option. Customers can then pick their preferred week for shipping! An example of how this could look is as follows:

    Local Delivery (Wednesdays) (Nov 23rd)
    Local Delivery (Wednesdays) (Nov 30th)
    Local Delivery (Wednesdays) (Dec 7th)

    The shipping date for whichever one of the shipping options they select - the “Instance of the method” they select - will be recorded alongside the standard details for the order.

    Information about the shipping date can be displayed in the method's title using “conversion specifiers”, as detailed in Appendix A.

    In fact, in this situation, it is essential to use conversion specifiers in the method's title as this is how the selected “instance” of the method - the selected week - is identified in the order's details.
