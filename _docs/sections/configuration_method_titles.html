Shipping Method Titles

Details of the options for method titles follow, the less frequently used Method Availability Scheduling and Method Shipping Scheduling options are described at a later stage.

Method Admin Title

This is the Title of the Shipping Method that will be displayed alongside the method's number in the admin only. It is intended simply to be used to aid in the configuration of the store's shipping methods.

For example, in a case where several products can be shipped using an “Overnight Ground” service but each product has an individual rate for that service:

Two separate methods may be defined, one for a “Dining Table” and one for a “Bookcase”.

The Method Title to be displayed to the customer for both should be “Overnight Ground”, but in the admin this would be confusing as two methods would have the same name in the Method Management Utility.

If the Admin Title of the first was set to:

“Overnight Ground - Dining Table”

and the second to:

“Overnight Ground - Bookcase”

..it would clearly be much easier to see which method was which, without having to examine each method's summary!

Method Title

This is the Title of the Shipping Method that will be displayed to the customer and recorded against the order once the checkout has been completed.

It is important that this is clear and concise as it is the only record the store owner will have of the method selected by the customer; it is the text used to identify this method in the order's details!

It is recommended that the name of the Shipping Service this method uses is part of the title!

HTML

The method title can make full use of HTML for more advanced layouts. For example, the title for a method, or its total cost, could be highlighted using <strong> tags.

Method Title Templates

The templates which are used to lay out the information about a shipping method use HTML and can be found in the file includes/languages/english/modules/shipping/advshipper.php. To change these templates, this file should be copied to the store's language templates folder and any changes made to this new file.

The hierarchy for these templates - that is, how each template is placed within any preceeding template - is clearly explained in the comments/documentation within that language file.

It is very simple but very flexible, allowing such things as creating a template to lay out information about a product which is being included in a method, with this template itself being repeated for each product that is included in the method.

Placement Tags

“Placement tags” can be used in the method title to position dynamic information about the respective shipping method. They can be entered directly in the method title or in the template.

Please Note: Once a method title uses any of these placement tags: {method_total}, {rate_calc_desc}, {surcharge_info}, {package_weights_desc} or {method_extra_title}, it no longer uses the default template and each piece of additional information will have to have a placement tag specified in the title, or a customised template, or it won't appear in the title!

The tags which can be used are:

  • {method_total} - This is the total cost for the method, formatted according to the current currency.

  • {rate_calc_desc} - This is a description of how the final rate was calculated. An example of its output is:

    1 x $2.00 + 2 x $1.50

    If multiple packages are required to ship the applicable products, because a Max Weight Per Package setting is in place and the applicable products (and any packaging added on for them) breach this weight limit, a breakdown of the packages will also be part of the rate description.

    An example of the output in a situation where the Max Weight Per Package is 50Kg, and the courier service charges a £0.15 per Kg for packages weighing 10-50Kg and a flat rate of £2.50 for packages under 10Kg, is:

    3 Packages: 50Kgs x £0.15/Kg + 43.5Kgs x £0.15/Kg + 8.9Kgs: £2.50

  • {surcharge_info} - This is the surcharge information for the method.

    • It can use a custom surcharge title, specified in the region's configuration.
    • If a custom title has not been specified, the default title specified in the language file is used.
    • The actual amount is formatted according to the current currency and placed wherever a {surcharge_amount} tag is found within the surcharge title.

    By allowing custom titles per region, more specific information can be provided to the customer.

    For example: A shipping method to a region could have “Handling Fee: {surcharge_amount}” while another might have “Fuel Surcharge: {surcharge_amount}”

  • {region_title} - This is the title of the region the customer's address is in (the region that is being shipped to). If the region has no title, the placement tag will be removed from the method title.

  • {package_weights_desc} - This is a description of the weights of the packages which would be sent via this shipping method. It is not intended to be used alongside any description of the rate calculations ({rate_calc_desc}) but instead of {rate_calc_desc}.

    It is designed to be used whenever it is desired to show how many packages the order contains, and what each weighs - if the corresponding tags ({package_weight}/{package_weights}) are used in the languages file - without displaying a breakdown of the rates for each package. An example of its output is:

    2 Packages: 145Kg + 65.35Kg

  • {method_extra_title} - When a shipping method is using a calculator, e.g. UPS/USPS, the exact name of the service being used is appended onto the method title by default. If this placement tag is used, this “Extra Title” is positioned in place of the tag, instead of simply being appended to the method title.

Date & Time “Placement Tags”

Special “Conversion Specifiers” can be used in the Title if the method has a value specified for its “Shipping Scheduling” to show information about the Date & Time that the method will be shipped. These are explained later in the section on Shipping Scheduling.

Method Descriptions

Zen Cart has very limiting restrictions on how shipping information can be formatted. It allows for just a single title assoicated with a cost. The ability of Advanced Shipper to use HTML and placement tags in titles can be used in roundabout way to add descriptions for a method. An example of how to add a description for a method can be found in the examples below.

Method Title Examples

  1. Example using the default method title template

    The standard template for method titles is as follows:

    {method_title} ({method_total})

  2. Example using the default method title template with Rate Calculation Descriptions

    If either the calculation method is “weight” or percentage or combination rates are used in the table of rates, and placement tags haven't been used in the method's title, the actual weight of the applicable products or the calculation used to come up with the final rate will be appended to the method's title using the template for rate calculation descriptions. The template for the method's title would then become:

    {method_title} ({method_total}) ({rate_calc_desc})

    For example, for a combination rate, this may result in the following:

    Royal Mail Next Day (£3.75) (3 x £1.25)

    As another example, if a simple rate was being used with a calculation method of weight, the following might be displayed:

    Royal Mail Next Day (£4.50) (12.25Kg)

  3. General Placement Tag Example

    Using placement tags in the title allows for much more fine-grained control over how the method title looks.

    For example, placement tags and HTML could be used to create the following title:

    Royal Mail Next Day <strong>({method_total})</strong>

    This would result in the total being highlighted but any rate calculations (or the weight of the applicable products if the calculation method was “weight”) wouldn't be displayed as the {rate_calc_desc} placement tag isn't present. An example of the output is as follows:

    Royal Mail Next Day (£4.50)

  4. Placement Tag with Surcharge Info Example

    As another example, if a region for a method had a Surcharge Title of “(Fuel Surcharge: {surcharge_amount})”, placement tags and HTML could be used to create the following title:

    Parcelforce Air <strong>({method_total})</strong> ({rate_calc_desc}) {surcharge_info}

    This would result in the surcharge being placed after the (highlighted) total. An example of the output is as follows:

    Parcelforce Air (£5.50) (1 x £4.50) (Fuel Surcharge: £1.00)

  5. Method Description Example (Uses Placement Tags)

    A description of the method can be added on a new line by preceeding it with a <br /> tag. If placement tags weren't used, the total would be appended to the description using the standard template, so obviously placement tags should always be used when adding a description!

    For example:

    <strong>Royal Mail Next Day</strong> ({method_total}) <br />Choose this method for the quickest delivery, normally before 12 noon.

    This would result in the following output:

    Royal Mail Next Day (£4.50)
    Choose this method for the quickest delivery, normally before 12 noon.
