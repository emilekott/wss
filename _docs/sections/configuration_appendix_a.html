Appendix A: Method Titles Conversion Specifiers

Information about the expected shipping date for a method can be given to the customer by putting special “conversion specifiers” in the method's title. These are simple character sequences which are substituted with date/time strings. The module simply uses PHP's strftime function to convert the expected shipping date/time into readable strings.

A full list of conversion specifiers can be found in the standard PHP documentation for the strftime function, but a short overview of some of the more useful specifiers is given here, along with a few examples of their use in method Titles:

%a - abbreviated weekday name according to the current locale
%A - full weekday name according to the current locale
%b - abbreviated month name according to the current locale
%B - full month name according to the current locale
%d - day of the month as a decimal number (range 01 to 31)
%H - hour as a decimal number using a 24-hour clock (range 00 to 23)
%M - minute as a decimal number
%r - time in a.m. and p.m. notation
%Y - year as a decimal number including the century

%a Delivery (%d %b from %H:%M)

For a method that is expected to be shipped on a Thursday at/from 9AM; may result in the title:

“Thurs Delivery (24 Nov from 09:00)”

%A Preferred Shipping (%B %d)

For a method that is expected to be shipped on a Wednesday; If used with a recurring method that shows a number of weeks of options in advance, it may result in the titles:

“Wednesday Preferred Shipping (November 23)”
“Wednesday Preferred Shipping (November 30)”
“Wednesday Preferred Shipping (December 07)”
