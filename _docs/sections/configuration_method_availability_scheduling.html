Method Availability Scheduling

Another powerful feature of the Advanced Shipper module is the ability to specify when the individual shipping methods should be available.

For example, a “Saturday Delivery” method could be configured to be available from Thursdays (at 16:00) until Fridays (at 16:00). The method would then only be available as an option from 4PM on Thursday afternoon until 4PM on Friday afternoon.

By default, methods have their Method Availability Scheduling set to “Always Available”.

Once Only

Setting the Method Availability Scheduling option to “Once Only” displays options which enable the shipping method to be made available for a single, specific time period only.

The value for Once Only Start Date/Time is optional and will default to the current date and time if not entered, but a value for Once Only End Date/Time must be specified. Both the Date and the Time must be specified. Their combined value is when this method will no longer appear as an option on the site.

Method Usage Limit

As well as setting a Date/Time when the method will no longer be an option, a limit on the number of orders which can make use of this method can be imposed by entering a value for the Method Usage Limit. If the number of orders which complete checkout matches this value before the Method's Once Only End Date/Time has passed, the Method will no longer appear as a shipping option on the site.

Recurring

Setting the Method Availability Scheduling option to “Recurring” displays options which enable the method to be made available for regular time periods.

Currently the only option for the Recurring Mode is “Weekly”. There are two main ways a method can be configured using this mode:

  1. Use A Weekly Start Day/Time And A Weekly Cut-off Day/Time

    Both a Weekly Start Day/Time and a Weekly Cut-off Day/Time can be specified, to limit the time period the method is available as a shipping option on a weekly basis.

    For example, to make a “Next Working Day” Method available only for Mondays - Thursdays (as deliveries made by the courier on weekends are charged differently):

    The Weekly Start Day/Time could be set to:

    Monday, 00:00

    and the Weekly Cut-off Day/Time could be set to:

    Thursday, 16:00.

    The method will then only be available as an option from first thing on Monday morning until 4PM on Thursday afternoon.

    As another example, to make a “Saturday Delivery” Method:

    The Weekly Start Day/Time could be set to:

    Thursday, 16:00

    and the Weekly Cut-off Day/Time to:

    Friday, 16:00.

    The method will then only be available as an option from 4PM on Thursday afternoon until 4PM on Friday afternoon.

  2. Use A Weekly Cut-off Day/Time Only

    Just the Weekly Cut-off Day/Time can be specified, enabling the creation of a method which repeats at the same day/time each week.

    This then enables the usage of the Number of Weeks to Show configuration option in the Shipping Scheduling section, which can make this method available for several weeks in advance, offering the customer the facility to pick their preferred week for shipping.

    This is described in more detail in the section on Shipping Scheduling. An example of the shipping options this could be used to create is as follows:

    Local Delivery (Wednesdays) (Nov 23)
    Local Delivery (Wednesdays) (Nov 30)
    Local Delivery (Wednesdays) (Dec 7)

Method Usage Limit

For either of these ways of configuring the “Recurring Weekly” Availability Scheduling, a limit on the number of orders which can make use of this method can be imposed by entering a value for the Method Usage Limit:

  1. If a Weekly Start Day/Time has been specified, then if the number of orders which complete checkout matches this value before the Method's Weekly Cut-off Day/Time has passed, the Method will be no longer appear as a shipping option on the site.

  2. If only the Weekly Cut-off Day/Time has been specified, then if the number of orders which complete checkout matches this value for any of the weeks the Method is available for, the Method will be no longer appear as a shipping option on the site for that week only.

Please note that the recording of method usage limits depends on their end timestamp, so no two methods should use the same shipping date and time for any given week!

Avoiding this problem is a simple matter of adjusting the shipping time by one second for each method (one ships at 16:00, next ships at 16:01 etc.).
