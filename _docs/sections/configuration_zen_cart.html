Standard Zen Cart Module Configuration

There aren't very many options in the standard Zen Cart shipping module configuration screen for this module and most are explained simply enough within that utility so aren't described here. Some may require more description though...

  • Compensate For Server Time

    All of the shipping methods' availability dates/times and shipping dates/times are taken based on the current server time. This should be checked/adjusted to match the current hour for the actual store. Otherwise it may appear as if the store is a different timezone from what the customer expects and they may not be shown accurate shipping options!

    Please Note: Not all stores will make use of this module's advanced method availability/shipping settings so this setting may not have any practical effect for your store.

  • Debugging Output

    As this module is so flexible, great care must be taken in creating a logical set of shipping options for a store. Sometimes mistakes are made/overlooked though, so the module can output some very useful debugging information when Debugging Output is enabled. This output will only work for the shipping stage of the checkout process; it cannot be used when moving on to the payment stage.

It is here also that error messages about the overall setup of the module will be displayed. This includes such things as the database tables being missing, but not errors or problems with individual configurations, they are displayed within the configuration utility.
