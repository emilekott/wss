  1. A JavaScript error message, “There was an error on this page. Error: missing ; before statement...” appears in the admin.

    This error occurs whenever the “Remote Loader” scripts can't run properly. More than likely you have the “Admin Profiles” module installed. Otherwise your admin session must have timed out.

    • Use the “Admin Profiles” utility to enable access to all of Advanced Shipper's admin files.

      This may take several jaunts into the Admin Profiles admin utility, as additional “locked out” files will become available to be “unlocked” as more of the Advanced Shipper Configuration utility is used.

  3. How can the overall title for the shipping be changed from the default of “Advanced Shipper” to something else?

    The text “Advanced Shipper” is used by default as Zen Cart requires that all shipping modules have something for their name. Changing it is easy enough but requires applying the same change each time the module is updated.

    To change the title, a copy of the language definition file for the module must be made for each language the store uses.

    As with all other language definition overrides in Zen Cart, this modified copy should be stored in the template directory for each language.

    • Copy the file includes/languages/english/modules/shipping/advshipper.php to the store's language templates folder. (E.g. includes/languages/english/modules/shipping/YOUR_TEMPLATE/advshipper.php).
    • Make the necessary changes to this copy of the file.

    Please Note: The module must have a title defined or it will not appear as an option at checkout! If it is preferred not to display a title for the module, please take a look at this FAQ.

    Please Note: Anytime the module is being updated to a newer version, the changes made in any copies of the languages file should be re-applied to fresh copies of the new language file for the updated version of Advanced Shipper. (I.e. this process should be repeated again after every upgrade).

  4. How can the overall title of the module and/or the brackets surrounding the shipping method quote(s)/title(s) be removed from the quote(s)/title(s)?

    Unfortunately the formatting/display of the overall quote(s)/title(s) cannot be controlled by Advanced Shipper. It is Zen Cart itself which controls this formatting/display.

    To change this formatting/display, changes must be made to core Zen Cart files.

    Important! Please Note: Below will follow some hints and two examples of how to make these changes. However, these changes are regarded by Ceon as “custom coding” and are therefore not covered by the free support offered with the purchase of the Advanced Shipper distribution from Ceon's website.

    If support from Ceon is required regarding these sorts of changes, payment to Ceon will be required to cover the time spent.

    Modifications introduction

    Zen Cart adds the module's title and the brackets to the quote(s)/title(s) in several files. Two of these files are discussed below. However, the store's Zen Cart installation may be customised and may have additional files which format/display this information. The format of the code - and subsequently the changes necessary - should be fairly similar in each file.

    includes/modules/pages/checkout_shipping/header_php.php

    The following line in this file adds the module's title to the formatting/display output, “$quote[0]['module'] .”, and adds brackets around the output from Advanced Shipper, “' (' .” and “. ')'”:

    'title' => (($free_shipping == true) ? $quote[0]['methods'][0]['title'] : $quote[0]['module'] . ' (' . $quote[0]['methods'][0]['title'] . ')'),

    To remove the module's title and the brackets from the display output, leaving only the actual shipping method quote(s), the line could be updated as follows:

    'title' => (($free_shipping == true) ? $quote[0]['methods'][0]['title'] : $quote[0]['methods'][0]['title']),

    includes/templates/template_default/templates/tpl_modules_shipping_estimator.php

    As with any other template file being modified for a store, the template file should copied from the template_default/templates folder to the template folder for the store.

    The following line in this file is one of several lines in the file which adds the module's title to the formatting/display output, “echo $quotes[$i]['module'];”, and adds brackets around the output from Advanced Shipper, “&nbsp;(” and “)”:

    <td><?php echo $quotes[$i]['module']; ?>&nbsp;(<?php echo $quotes[$i]['methods'][0]['title']; ?>)</td>

    To remove the module's title and the brackets from the display output, leaving only the actual shipping method quote(s), the line could be updated as follows:

    <td><?php echo $quotes[$i]['methods'][0]['title']; ?></td>

    The other lines of the file which also add the module's title and brackets to the display output are very similar so will not be discussed here.

  5. How can the title for the shipping method selected by the customer be simplified in the Order Total listings and/or the Order e-mail?

    Unfortunately the formatting/display title for the shipping method selected by the customer cannot be controlled by Advanced Shipper. It is Zen Cart itself which controls this formatting/display.

    To change this formatting/display, changes must be made to core Zen Cart files.

    Important! Please Note: Below will follow some hints and two examples of how to make these changes. However, these changes are regarded by Ceon as “custom coding” and are therefore not covered by the free support offered with the purchase of the Advanced Shipper distribution from Ceon's website.

    If support from Ceon is required regarding these sorts of changes, payment to Ceon will be required to cover the time spent.

    Modifications introduction

    Zen Cart formats the full title of the shipping method selected by the customer for display in several files. Three of these files are discussed below. Several additional files within the standard Zen Cart distribution also perform this formatting but are not discussed here as the process for modifying them is quite similar. The store's Zen Cart installation may also be customised and may have further additional files which format/display this information. The format of the code - and subsequently the changes necessary - is likely to be fairly similar in each file.

    Order Total Summary Displays within the Store

    File: includes/templates/template_default/templates/tpl_modules_order_totals.php

    As with any other template file being modified for a store, the template file should copied from the template_default/templates folder to the template folder for the store.

    The following lines in this file output the Order Total information:

    for ($i=0; $i<$size; $i++) { ?>
<div id="<?php echo str_replace('_', '', $GLOBALS[$class]->code); ?>">
    <div class="totalBox larger forward"><?php echo $GLOBALS[$class]->output[$i]['text']; ?></div>
    <div class="lineTitle larger forward"><?php echo $GLOBALS[$class]->output[$i]['title']; ?></div>
</div>
<br class="clearBoth" />
<?php } ?>

    To simplify the output for the shipping info, a check needs to be made against which order total module is currently having its output formatted for display and the desired adjustments made. In this case, a title of “Shipping” is used for the Order Total line if it is the line concerned with outputting the shipping information:

    for ($i=0; $i<$size; $i++) { ?>
<div id="<?php echo str_replace('_', '', $GLOBALS[$class]->code); ?>">
    <div class="totalBox larger forward"><?php
        if ($GLOBALS[$class]->code == 'ot_shipping') {
                echo 'Shipping';
        } else {
                echo $GLOBALS[$class]->output[$i]['text'];
        }
        ?></div>
    <div class="lineTitle larger forward"><?php echo $GLOBALS[$class]->output[$i]['title']; ?></div>
</div>
<br class="clearBoth" />
<?php } ?>

  6. The message “Sorry but there are no shipping options to your address at this time!” appears.

    This message is displayed because one or more of the item(s) in the cart doesn't have a shipping method set up which has a table of rates that covers the item(s), either on their own, or in the combination in which they are present in the cart.

    This situation can only be avoided by careful planning of the shipping methods, regions and rates.

    Does the table of rates that covers the item(s) have a provision for the number of item(s) present in the cart? Or for the combined weight of the item(s) in the cart? Care must be taken to check that a simple logical error hasn't been made in the configuration!

  8. The error “Module configuration error: The Store's Postcode is not valid!” appears.

    For the geolocation functionality to work it must know where your store is located!

    You have to ensure that your store's shipping address country and postcode have been set correctly in the Zen Cart admin.

    You can change these settings in Configuration > Shipping/Packaging.

  9. Does this module use the store's Zone Definitions?

    No, this module only makes use of the country for a shipping address and, optionally, the postcode for that address.

    If the module has been configured to use geolocation it will always require a postcode to check against, the same applies if postcode ranges have been specified in any of the regions' Countries/Postcodes configuration values.

    If a customer has not entered their postcode properly the module will warn them. It will even display instructions on how to update their postcode, with specific information about the format of postcodes for their country!

    The module can be extended to use Zones using the Advanced Shipper Zones expansion module, and/or even more options, Country/State Matching and Country/City Matching can be added using the Advanced Shipper Localities expansion module.

    Please Note: Advanced Shipper Localities is not going to be upgraded for Zen Cart 2.x. (Advanced Shipper and Advanced Shipper Zones both will be though).

  10. The rate being shown to the customer is several times the expected rate!

    Zen Cart has a configuration setting which can affect how a weight-based order is shipped (outside of Advanced Shipper's own Maximum Weight Per Package setting).

    • Check the setting for “Enter the Maximum Package Weight you will ship” in Configuration > Shipping/Packaging.

      If it is too low the customer's order will be split into several deliveries, before Advanced Shipper has a chance to build its own quotes. It should be raised to an appropriate (i.e. very high) value, allowing Advanced Shipper to handle the shipping for the order properly!

  12. I'm not sure how to configure regions. Help!

    The best way to learn about configuring this module is to read the documentation. Some insight into how to define regions can be gained from taking a look at the example region definitions in Appendix B.

    However, once you've done that, the main point to remember is that the module identifies the region a customer's address is in on a “first region to match” basis, so great care has to be taken in the planning of the order of the region definitions. The first regions should have the most specific definitions, allowing the latter regions to act as “fall backs” which catch any addresses that are not covered by any of the previous regions.

  13. The module isn't showing in the list of shipping modules in the admin.

    If this is happening, the first thing that should be checked is if there are two advshipper.php files in includes/modules/shipping/.

    If you have backed up a previous version of Advanced Shipper in that folder, under a different name, Zen Cart will fail! Zen Cart auto-instantiates every file in the includes/modules/payment / shipping / order_total directories, so when you create a copy of a file in any of those directories, there are two files declaring the same class, which causes PHP to fail with a fatal error (which Zen Cart normally doesn't display!).

    • Delete the duplicate/renamed file to fix this problem. There should only be one Advanced Shipper file in includes/modules/shipping.

    If the above doesn't apply, the only other option is that the files haven't uploaded properly.

    • Re-upload the files from the module's distribution, overwriting all existing files.

  14. A country I'd like to check postcode ranges against isn't covered, what can I do?

    This module is extensible and will automatically try to run a postcode matching method for any country code used in a Region X Countries/Postcodes configuration field.

    The method called has to have a name in the format “_regionMatchesRangeXX()” where ‘XX’ is the two digit ISO code for the country.

    If you are handy with PHP, just look at the existing methods and the format of the postcodes for the country in question and add this method yourself! :)

    Of course, it'd be appreciated by all if, having done so, you would send the code to us so we can add it to the module for others to use.

    Please Note: If you want us to write a method to match postcodes for a country for you then please be aware that we will charge for this service. The rate is £40 an hour with a minimum charge of £60. Examining the postcode formats for an entire country and coding a routine to match postcodes for that country takes a reasonable amount of effort!
