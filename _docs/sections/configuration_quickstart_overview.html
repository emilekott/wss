Configuration Overview/Quickstart

Configuration of the Shipping Methods and Regions for the store takes place with the Advanced Shipper Configuration Utility. This can be accessed by going to:

  • Modules > Advanced Shipper Configuration

Mirroring Real Life

When sending products, a store usually has several different shipping services available to them. The differences between the services could be the speed of delivery for items in that delivery, or a difference in the way a delivery is shipped (and therefore its cost). These different services could apply to all products or just specific products and/or categories and/or products from a specific manufacturer.

Advanced Shipper works in much the same way as the real-life shipping options available to the store:

  1. A Shipping Method is added in the configuration.

    A title is given to the method to identify it, e.g. “Royal Mail 2nd Class” or “Royal Mail Next Day”.

  2. The Products this Method applies to are selected.

    Particular methods may apply only for certain products, e.g. books may be sent using a standard postal service while bookcases may have to be sent using an overnight courier service. By selecting specific products and/or categories and/or manufacturers a method can be restricted to being applicable only for those products/categories/manufacturers.

  3. Individual Tables of Rates are entered for each Region the Method covers.

    Most shipping methods have differing prices depending on the address the delivery is being sent to. Each method can have any number of regions defined, each of which can have a table of rates which is used to work out how much delivery should cost for the product(s) being ordered to be delivered to the customer's address.

Additional Method Options

Individual shipping methods can be set to be available only for certain time periods, preferred shipping dates can be offered to the customer or a limit on the number of orders which can make use of a particular method within its availability period can be set using the Method Availability Scheduling and Method Shipping Scheduling options for each and every method in the configuration.

Checking/Debugging Configurations

The module has a very handy Debugging Output option in its Standard Zen Cart Module Configuration settings. It can be used on the store side, to check that the methods for the store have been set up properly for different combinations of cart contents.

It is vital that the combination of methods defined for the store cover all products, all supported regions to be delivered to and all supported total weight/total price/total number of items limits... it is up to whoever is configuring the store to make sure that the combination of methods they have set up make practical, logical sense!

As detailed in the Support section, we can help with simple configuration questions, but complex configuration questions can require lots of time to examine, so we have to bill for the time spent... the Debugging Output should make it easier to spot mistakes/omissions which result in “No Shipping Options“ messages on the site!

Don't forget to check the FAQs as well for possible solutions to any problems!
