Appendix B: Example Region Definitions for Various Countries

The following example Postcode Range Definitions for each country supported by this module should hopefully make it easier to work out how to configure the module. Where possible a link to a list of postcode ranges for the country in question has been added. As these links are to third party sites we can accept no responsibility for their accuracy, but hopefully they are of use...

Please remember that the most specific region definitions should come first, followed by the more general “catch-all” definitions.

Australia

Australian postcodes are numeric, consisting of three or four digits.

Example: To set up a region for the City of “Brisbane”, a region for the state of “Queensland” and a region for the “Rest of Australia”, three regions with the following Countries/Postcodes configuration values would be added:

  • Region 1: AU:4000

    Region 2: AU:4000-4999,AU:9000-9999 (Covers 4002, 4013, 4902, 9001 etc.)

    Region 3: AU (Covers all other states and cities in Australia)

A list of postcodes for states and cities in Australia can be found in this wikipedia article.

The Region Titles for the above regions could be as follows:

Region 1: “City of Brisbane”
Region 2: “State of Queensland”
Region 3: “Rest Of Australia”

Canada

Canadian postcodes are of the format X9X 9X9, where X is a letter and 9 is a digit, with a space separating the third and fourth characters. An example is K1A 0B1, which is for Canada Post's Ottawa headquarters.

The first three characters are known as the “Forward Sortation Area” (FSA), the last three as the “Local Delivery Unit”. Only the FSA is used by this module.

Example: To set up a region for “St John's in Newfoundland”, a region for the province of “Ontario” and a region for the “Rest of Canada”, three regions with the following Countries/Postcodes configuration values would be added:

  • Region 1: CA:A1A-S (Covers A1A 0E8, A1C 3E4, A1S 8C9 etc. but not A2A 0E8 or A1T 8C9 etc.)

    Region 2: CA:P (Covers P0V 5C6, P3Y 7C2, P8N 6T6 etc.)

    Region 3: CA (Covers all other provinces and cities in Canada)

An extremely comprehensive breakdown of all the FSAs in Canada can be found in this PDF document.

The Region Titles for the above regions could be as follows:

Region 1: “St John's, Newfoundland”
Region 2: “Ontario”
Region 3: “Rest Of Canada”

France

France uses five-digit numeric postcodes, the first two digits normally representing the “Département”. In Paris, the last two digits indicate the “arrondissement”.

Example: To set up a region for the “8th arrondissement in Paris”, a region for the city of “Paris”, a region for the region of “Bretagne” (regional diagram) and a region for the “Rest of France”, four regions with the following Countries/Postcodes configuration values would be added:

  • Region 1: FR:75008 (Covers 75008 only!)

    Region 2: FR:75 (Covers 75000, 75007, 75116 etc.)

    Region 3: FR:29, FR:22, FR:35, FR:56 (Covers 29000, 22160, 56008 etc.)

    Region 4: FR (Covers all other Départements and cities in France)

A reasonably complete list of postcodes for Départements in France can be found in this wikipedia article. At least one Département, the island of Corsica (postcodes beginning ‘20’) isn't covered in this article, so more may not be either.

The Region Title s for the above regions could be as follows:

Region 1: “8th Arrondissement, Paris”
Region 2: “Paris”
Region 3: “Bretagne”
Region 4: “Rest Of France”

Ireland

Most places in Ireland, rather unhelpfully, don't use postcodes. Only Dublin has “Postal Districts”. Therefore, Countries/Postcodes matching can only be used for matching postal districts within Dublin.

Example: To set up a region for the postal district of “Dublin 1”, a region for the city of “Dublin” and a region for the “Rest of Ireland”, three regions with the following Countries/Postcodes configuration values would be added:

  • Region 1: IE:Dublin 1

    Region 2: IE:Dublin (Covers Dublin 1, Dublin 4, Dublin 15 etc.)

    Region 3: IE (Covers all other cities in Ireland)

Information on Dublin's postal districts can be found in this wikipedia article.

The Region Titles for the above regions could be as follows:

Region 1: “Dublin 1”
Region 2: “Dublin”
Region 3: “Rest Of Ireland”

Poland

Polish postcodes are numeric, consisting of five digits, normally with a dash/hyphen separating the second and third digits, splitting the postcode into two groups.

Please Note: The hyphen is removed from the postcodes before they are compared, the comparison takes place as if the postcode was a 5 digit number.

Example: To set up a region for the Town of “Dębno”, a region for the City of “Kraków” and a region for the “Rest of Poland”, three regions with the following Countries/Postcodes configuration values would be added:

  • Region 1: PL:34434 (Covers 34-434)

    Region 2: PL:30024-31962 (Covers 30-024, 30-634, 31-840 etc.)

    Region 3: PL (Covers all other towns/cities and postal districts in Poland)

A list of postcodes in Poland can be found in this wikipedia article.

The Region Titles for the above regions could be as follows:

Region 1: “Town of Dębno”
Region 2: “City of Kraków”
Region 3: “Rest Of Poland”

Example language definitions for the above regions could be as follows:

The UK

UK postcodes are of the format X9 9XX, X99 9XX, X9X 9XX, XX9 9XX, XX99 9XX or XX9X 9XX, where X is a letter and 9 is a digit. An example is BT10 0JX.

The first few characters before the space are known as the “Outbound Code”, the characters after the space as the “Inbound Code”. Only the Outbound Code is used by this module.

Example: To set up a region for “Northern Ireland”, a region for “The Isle of Man, the Scilly Isles & the Channel Islands” and a region for the “Rest of the UK”, three regions with the following Countries/Postcodes configuration values would be added:

  • Region 1: GB:BT (Covers BT1 3XF, BT10 0JX, BT27 4FT etc. but not BS1 3XF or B2 3DF etc.)

    Region 2: GB:IM,GB:GY,GB:JE,GB:TR21-25 (Covers IM3 4FR, GY4 7DS, TR21 4FD etc. but not TR2 3LF or TR26 7HG etc.)

    Region 3: GB (Covers all other postcode areas in the UK)

A (slightly unwieldy) list of postcodes in the UK can be found in this wikipedia article.

The Region Titles for the above regions could be as follows:

Region 1: “Northern Ireland”
Region 2: “The Isle of Man, the Scilly Isles & the Channel Islands”
Region 3: “UK Mainland”

The US

US postcodes are numeric, consisting of five digits. There is also an extended version, called "ZIP+4", which adds a hyphen and four additional digits following the main ZIP code, to identify a smaller geographical area or single large entity. Only the first 5 digits are made use of by this module.

Example: To set up a region for the City of “Seattle”, a region for the state of “Washington” and a region for the “Rest of the US”, three regions with the following Countries/Postcodes configuration values would be added:

  • Region 1: US:981 (Covers 98102, 98152, 98199 etc. but not 98012 or 98200 etc.)

    Region 2: US:980-994 (Covers 98000, 98102, 99450 etc. but not 99500 or 99655 etc.)

    Region 3: US (Covers all other states and cities in US)

There are quite a few lists of US Zip codes. A map showing zip code ranges for all the individual states (inc Hawaii and Alaska) can be found in this wikipedia article.

The Region Titles for the above regions could be as follows:

Region 1: “Seattle”
Region 2: “Washington State”
Region 3: “Rest Of The US”
