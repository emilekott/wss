System Requirements

  • Javascript is required for the Configuration Utility. Browsers tested include Firefox 2.x, Firefox 3.x and IE 7.0.

Installation

Installation of the module is plain and simple as it only requires the files in the contribution archive to be copied across and the database script to be run.

Installation Overview

  1. Copy the files from the distribution into the Zen Cart installation.
  2. Create the necessary database tables and update other tables by running the provided SQL script.
  3. [Optional] Create any geolocation database(s) applicable for the store by running the provided SQL script(s).
  4. Install the Advanced Shipper module.
  5. Configure the Advanced Shipper module.

Installation Steps

  1. Copy the Advanced Shipper module's files to the Zen Cart installation directory

    These are the files which make up this module. They should be copied into their respective folders in the Zen Cart installation.

    They already have the correct folder structure, so a single drag-and-drop of the folders/files in the files folder into the root directory for the store will put the files in the correct locations; there's no need to copy the files individually - that often results in files being missed out!

    Please Note: If the admin folder for the store is not named “admin” then care must be taken to place the files from within the files/admin folder into the store's admin folder, copying across the files/admin folder won't work!

  2. Create the necessary Database Tables and Update Other Tables

    A SQL script, advshipper.sql, is provided with this module to create the database tables the module uses. It will also modify two of the existing tables, orders and orders_total, so that they can store the longer shipping method titles this module can generate. This will not affect your shop in any negative way!

    • You can go to Tools > Install SQL Patches in the Zen Cart Admin and run the file through that.

    • Or software such as PHPMyAdmin can be used to run the SQL script (Care must be taken to check the name of the tables. If the store uses a prefix for the table names it is essential that this prefix is put in front of the table names). If using this method please note that some double backslashes may need to be removed in the source.

  3. [OPTIONAL] Create Geolocation Database(s).

    This module is completely dynamic in terms of its geolocation support, so some stores may need to create one or more database tables for use in identifying the location of a shipping address in relation to the store (and the location of the store itself for that matter!).

    At the time of writing, the module comes with three SQL files for the creation of geolocation databases for the UK, the US and Australia. It is anticipated that more will be added as time goes on. The SQL scripts for the UK, uk_postcodes.sql, the US, us_zipcodes.sql, and Australia, au_postcodes.sql, can be found in the geolocation_db_tables folder.

    • You can go to Tools > Install SQL Patches in the Zen Cart Admin and run the file through that

    • Or software such as PHPMyAdmin can be used to run the SQL script (Care must be taken to check the name of the tables. If the store uses a prefix for the table names it is essential that this prefix is put in front of the table names). If using this method please note that some double backslashes may need to be removed in the source.

  4. Install the Advanced Shipper module

    Installing the module is just as simple as with any other Zen Cart module.

    • Go to Modules > Shipping, click on advshipper and then on Install

  5. Configure the Advanced Shipper module

    Now all that has to be done is to configure the module in the Zen Cart Admin. The powerful configuration utility is accessed by going to Modules > Advanced Shipper Config.
